BUILDAR:
docker build -f docker/Dockerfile -t llm-annotation:latest .


SUBIR OS CONTAINERS:
# Para cada container precisa de uma OLLAMA_PORT diferente, senao da conflito no host.
GPU_IDS=1 CUDA_DEVICE_IDS=0 PORT=8000 OLLAMA_PORT=11434 docker compose -p gpu1 -f docker/docker-compose.yml up -d
GPU_IDS=2 CUDA_DEVICE_IDS=0 PORT=8001 OLLAMA_PORT=11435 docker compose -p gpu2 -f docker/docker-compose.yml up -d

DERRUBALOS:
docker compose -p gpu1 -f docker/docker-compose.yml down
docker compose -p gpu2 -f docker/docker-compose.yml down

LOGS:
docker logs -f gpu1-llm-annotation-1
docker logs -f gpu2-llm-annotation-1


==================================================================
PRIMEIRO START — DEMORA!
==================================================================
Na primeira vez que cada container sobe, o entrypoint:
  1. Sobe `ollama serve` com OLLAMA_NUM_PARALLEL=5, FLASH_ATTENTION=1,
     KV_CACHE_TYPE=q8_0, CONTEXT_LENGTH=4096, KEEP_ALIVE=24h.
  2. Faz `ollama pull` dos 3 modelos (~15 GB no total):
       - qwen3:8b
       - llama3.1:8b
       - deepseek-r1:8b
  3. Pre-carrega os 3 modelos na VRAM (POST em /api/generate).
  4. Sobe a API FastAPI em :8000.

Os modelos ficam salvos no volume nomeado `ollama_models` e persistem
entre restarts/rebuilds. As proximas subidas pulam o download e so
fazem o preload em VRAM (rapido).

Acompanhar:
  docker logs -f gpu1-llm-annotation-1


==================================================================
LIMPAR OS MODELOS BAIXADOS (forca re-download)
==================================================================
docker volume rm llm-annotation_ollama_models
